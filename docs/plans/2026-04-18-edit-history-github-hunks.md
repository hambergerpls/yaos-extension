# Edit History GitHub-Style Hunk Rendering Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the current "all spans flowed inline" rendering (for stored diffs and mid-chain synthetics) with GitHub-style row-by-row hunks. Each logical line is its own block with a `+`/`-`/` ` gutter. Unchanged lines outside a 3-line context window collapse behind a simple `… N unchanged lines` separator. Initial-snapshot path and `(diff unavailable)` fallback remain unchanged.

**Architecture:** Two new pure functions in `editHistory/editHistoryDiff.ts` (`segmentLines`, `buildHunks`) do all the non-DOM work. A new private `renderHunks` method on `EditHistoryView` replaces `renderDiffOps` at the two diff-producing call-sites. The initial-snapshot path still uses `renderDiffOps` on its synthesized `[[1, truncated]]` op, so the single-span initial rendering is preserved. CSS gets new `.diff-hunk`, `.diff-line`, `.diff-gutter`, `.diff-line-text`, `.diff-add-line`, `.diff-del-line`, `.diff-retain-line`, and `.diff-hunk-skip` classes (all prefixed `yaos-extension-edit-history-`). The previous `.diff-add`/`.diff-del`/`.diff-retain` classes remain in CSS for the initial-snapshot path.

**Tech Stack:** TypeScript, `fast-diff` (existing — no new library), Vitest + happy-dom for DOM assertions, esbuild for the bundle.

## The `segmentLines` algorithm

Walk the ops maintaining two running buffers, `oldLineBuf` and `newLineBuf`, and emit rows when a `\n` is encountered or the op stream ends.

```
for each op [opCode, text]:
  if opCode === 0:       # retain — appears in both sides
    for each char c in text:
      if c === "\n":
        flushLine(oldLineBuf, newLineBuf) -> emits rows
        oldLineBuf = ""; newLineBuf = ""
      else:
        oldLineBuf += c; newLineBuf += c
  elif opCode === 1:     # add — only new side
    for each char c in text:
      if c === "\n":
        flushLine(oldLineBuf, newLineBuf)
        oldLineBuf = ""; newLineBuf = ""
      else:
        newLineBuf += c
  elif opCode === -1:    # del — only old side
    for each char c in text:
      if c === "\n":
        flushLine(oldLineBuf, newLineBuf)
        oldLineBuf = ""; newLineBuf = ""
      else:
        oldLineBuf += c

at end:
  if oldLineBuf !== "" or newLineBuf !== "":
    flushLine(oldLineBuf, newLineBuf)
```

**`flushLine(old, new)`:**

```
if old === new:                      # pure retain line (or empty-empty after trailing \n)
  if old !== "": emit { kind: "retain", text: old }
elif old === "" and new !== "":      # pure insert line
  emit { kind: "add", text: new }
elif old !== "" and new === "":      # pure delete line
  emit { kind: "del", text: old }
else:                                # paired edit — both rows
  emit { kind: "del", text: old }
  emit { kind: "add", text: new }
```

**Key property:** pure-retain inputs (`[[0, "a\nb\nc"]]`) produce only retain rows. Pure-insert inputs produce only add rows. The order of emitted rows follows the source text reading order, with the convention that a paired edit emits del-row before add-row.

## The `buildHunks` algorithm

Given a `DiffLine[]` and `context: number`:

1. Collect indices of change lines: `changeIdx = [i | lines[i].kind !== "retain"]`.
2. If `changeIdx.length === 0`: return `[]`.
3. Build a boolean `keep[i]` array of length `lines.length`, initially all `false`.
4. For each `i` in `changeIdx`: for `j` in `[max(0, i-context), min(lines.length-1, i+context)]`: `keep[j] = true`.
5. Walk `keep` left-to-right. Track segments:
   - Run of `keep[i] === true` → flush as `{ kind: "hunk", lines: lines.slice(start, end+1) }`.
   - Run of `keep[i] === false` → flush as `{ kind: "skip", count: end - start + 1 }`.
6. Emit the items in order encountered.

**Edge cases:**
- Leading `false` run (retains at start that fall outside any context window) → emitted as a `skip` item at position 0.
- Trailing `false` run → emitted as a `skip` item at the end.
- Two changes with context windows that overlap or touch → one merged `hunk`.
- Two changes whose windows don't overlap → two hunks separated by a `skip`.

```ts
export type DiffLine =
	| { kind: "retain"; text: string }
	| { kind: "add"; text: string }
	| { kind: "del"; text: string };

export function segmentLines(ops: DiffOp[]): DiffLine[] {
	const result: DiffLine[] = [];
	let oldBuf = "";
	let newBuf = "";

	const flush = () => {
		if (oldBuf === newBuf) {
			if (oldBuf !== "") result.push({ kind: "retain", text: oldBuf });
		} else if (oldBuf === "") {
			result.push({ kind: "add", text: newBuf });
		} else if (newBuf === "") {
			result.push({ kind: "del", text: oldBuf });
		} else {
			result.push({ kind: "del", text: oldBuf });
			result.push({ kind: "add", text: newBuf });
		}
		oldBuf = "";
		newBuf = "";
	};

	for (const [op, text] of ops) {
		for (const ch of text) {
			if (ch === "\n") {
				flush();
				continue;
			}
			if (op === 0) {
				oldBuf += ch;
				newBuf += ch;
			} else if (op === 1) {
				newBuf += ch;
			} else if (op === -1) {
				oldBuf += ch;
			}
		}
	}
	flush();

	return result;
}
```

```ts
export type HunkItem =
	| { kind: "hunk"; lines: DiffLine[] }
	| { kind: "skip"; count: number };

export const DEFAULT_CONTEXT_LINES = 3;

export function buildHunks(lines: DiffLine[], context: number): HunkItem[] {
	const changeIdx: number[] = [];
	for (let i = 0; i < lines.length; i++) {
		if (lines[i]!.kind !== "retain") changeIdx.push(i);
	}
	if (changeIdx.length === 0) return [];

	const keep = new Array<boolean>(lines.length).fill(false);
	for (const i of changeIdx) {
		const lo = Math.max(0, i - context);
		const hi = Math.min(lines.length - 1, i + context);
		for (let j = lo; j <= hi; j++) keep[j] = true;
	}

	const items: HunkItem[] = [];
	let i = 0;
	while (i < lines.length) {
		if (keep[i]) {
			const start = i;
			while (i < lines.length && keep[i]) i++;
			items.push({ kind: "hunk", lines: lines.slice(start, i) });
		} else {
			const start = i;
			while (i < lines.length && !keep[i]) i++;
			items.push({ kind: "skip", count: i - start });
		}
	}
	return items;
}
```

```ts
	private renderHunks(container: HTMLElement, ops: DiffOp[]): void {
		const lines = segmentLines(ops);
		const items = buildHunks(lines, DEFAULT_CONTEXT_LINES);
		for (const item of items) {
			if (item.kind === "skip") {
				const skipEl = container.createDiv({ cls: "yaos-extension-edit-history-diff-hunk-skip" });
				skipEl.textContent = `… ${item.count} unchanged lines`;
				continue;
			}
			const hunkEl = container.createDiv({ cls: "yaos-extension-edit-history-diff-hunk" });
			for (const line of item.lines) {
				let lineCls: string;
				let gutterChar: string;
				if (line.kind === "retain") {
					lineCls = "yaos-extension-edit-history-diff-retain-line";
					gutterChar = " ";
				} else if (line.kind === "add") {
					lineCls = "yaos-extension-edit-history-diff-add-line";
					gutterChar = "+";
				} else {
					lineCls = "yaos-extension-edit-history-diff-del-line";
					gutterChar = "-";
				}
				const rowEl = hunkEl.createDiv({ cls: `yaos-extension-edit-history-diff-line ${lineCls}` });
				const gutter = rowEl.createSpan({ cls: "yaos-extension-edit-history-diff-gutter" });
				gutter.textContent = gutterChar;
				const textEl = rowEl.createSpan({ cls: "yaos-extension-edit-history-diff-line-text" });
				textEl.textContent = line.text;
			}
		}
	}
```

```markdown
GitHub's default.
```

```markdown
below its summary line. Versions with stored `diff` ops and mid-chain
rebase bases (`content`-only versions at `versionIndex > 0` that synthesize
```
